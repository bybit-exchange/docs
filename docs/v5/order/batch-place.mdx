> **Covers: Spot (UTA, UTA Pro) / Option (UTA, UTA Pro) / USDT Perpetual, USDC Perpetual, USDC Futures (UTA Pro)**

:::tip
This endpoint allows you to place more than one order in a single request.
- Make sure you have sufficient funds in your account when placing an order. Once an order is placed, according to the
funds required by the order, the funds in your account will be frozen by the corresponding amount during the life cycle
of the order.
- A maximum of 20 orders (option) & 10 orders (linear) & 10 orders (spot) can be placed per request. The returned data list is divided into two lists.
The first list indicates whether or not the order creation was successful and the second list details the created order information. The structure of the two lists are completely consistent.
:::

:::info
- Check the rate limit instruction when category=`linear` or `spot` [here](../rate-limit#api-rate-limit-rules-for-vipspros)

- **Risk control limit notice:**<br/>
Bybit will monitor on your API requests. When the total number of orders of a single user (aggregated the number of orders across main account and sub-accounts) within a day (UTC 0 - UTC 24) exceeds a certain upper limit, the platform will reserve the right to remind, warn, and impose necessary restrictions.
Customers who use API default to acceptance of these terms and have the obligation to cooperate with adjustments.
:::

### HTTP Request
POST `/v5/order/create-batch`

### Request Parameters

|request |**true** |array |Object |

|> side |**true** |string |`Buy`, `Sell`|

|> marketUnit |false |string |The unit for `qty` when create **Spot market** orders for **UTA account** <li>`baseCoin`: for example, buy BTCUSDT, then "qty" unit is BTC</li><li>`quoteCoin`: for example, sell BTCUSDT, then "qty" unit is USDT</li>|
|> price |false |string |Order price <ul><li>Market order will ignore this field</li><li>Please check the min price and price precision from [instrument info](../market/instrument#response-parameters) endpoint</li><li>If you have position, price needs to be better than liquidation price</li></ul> |
|> triggerDirection |false |integer |Conditional order param. Used to identify the expected direction of the conditional order. <ul><li>`1`: triggered when market price rises to `triggerPrice`</li><li>`2`: triggered when market price falls to `triggerPrice`</li> </ul>Valid for `linear`|
|> orderFilter |false |string |If it is not passed, `Order` by default. <ul><li>`Order`</li><li>`tpslOrder`: Spot TP/SL order, the assets are occupied even before the order is triggered</li><li>`StopOrder`: Spot conditional order, the assets will not be occupied until the price of the underlying asset reaches the trigger price, and the required assets will be occupied after the Conditional order is triggered</li></ul>Valid for `spot` **only**|
|> triggerPrice |false |string |<ul><li>For Perps & Futures, it is the conditional order trigger price. If you expect the price to rise to trigger your conditional order, make sure:<br/> _triggerPrice > market price_<br/>Else, _triggerPrice < market price_</li><li>For spot, it is the orderFilter=tpslOrder, or orderFilter=stopOrder trigger price</li></ul>|
|> [triggerBy](../enum#triggerby) |false |string |Conditional order param (Perps & Futures). Trigger price type. `LastPrice`, `IndexPrice`, `MarkPrice`|
|> orderIv |false |string |Implied volatility. `option` **only**. Pass the real value, e.g for 10%, 0.1 should be passed. `orderIv` has a higher priority when `price` is passed as well |
|> [timeInForce](../enum#timeinforce) |false |string |<a href="https://www.bybit.com/en-US/help-center/bybitHC_Article?language=en_US&id=000001044">Time in force</a> <ul><li>Market order will use `IOC` directly</li><li>If not passed, `GTC` is used by default</li> </ul>|
|> [positionIdx](../enum#positionidx) |false |integer |Used to identify positions in different position modes. Under hedge-mode, this param is **required** (USDT perps have hedge mode)<ul><li>`0`: one-way mode</li><li>`1`: hedge-mode Buy side</li><li>`2`: hedge-mode Sell side</li></ul>|
|> orderLinkId |false |string |User customised order ID. A max of 36 characters. Combinations of numbers, letters (upper and lower cases), dashes, and underscores are supported.<br/>_Futures, Perps & Spot: orderLinkId rules_:<br/><ul><li>optional param</li><li>always unique</li>_`option` orderLinkId rules_:<br/><li>**required** param</li><li>always unique</li></ul>|
|> takeProfit |false |string |Take profit price, vaild for `linear`, `spot`<li>`spot(UTA)`: Spot Limit order supports take profit, stop loss or limit take profit, limit stop loss when creating an order</li>|
|> stopLoss |false |string |Stop loss price, vaild for `linear`, `spot` <li>`spot(UTA)`: Spot Limit order supports take profit, stop loss or limit take profit, limit stop loss when creating an order</li>|
|> [tpTriggerBy](../enum#triggerby) |false |string |The price type to trigger take profit. `MarkPrice`, `IndexPrice`, default: `LastPrice`.<br/>Valid for `linear` |
|> [slTriggerBy](../enum#triggerby) |false |string |The price type to trigger stop loss. `MarkPrice`, `IndexPrice`, default: `LastPrice`<br/>Valid for `linear` |
|> reduceOnly |false |boolean |<a href="https://www.bybit.com/en-US/help-center/bybitHC_Article?language=en_US&id=000001047">What is a reduce-only order?</a> `true` means your position can only reduce in size if this order is triggered. <ul><li>You **must** specify it as `true` when you are about to close/reduce the position</li><li>When reduceOnly is true, take profit/stop loss cannot be set</li></ul>Valid for `linear`, & `option` |
|> closeOnTrigger |false |boolean |<a href="https://www.bybit.com/en-US/help-center/bybitHC_Article?language=en_US&id=000001050">What is a close on trigger order?</a> For a closing order. It can only reduce your position, not increase it. If the account has insufficient available balance when the closing order is triggered, then other active orders of similar contracts will be cancelled or reduced. It can be used to ensure your stop loss reduces your position regardless of current available margin.<br/>Valid for `linear`|
|> [smpType](../enum#smptype) |false |string |Smp execution type. [What is SMP?](../smp)|
|> mmp |false |boolean |Market maker protection. `option` **only**. `true` means set the order as a market maker protection order. [What is mmp?](../account/set-mmp) |
|> tpslMode |false |string |TP/SL mode <ul><li>`Full`: entire position for TP/SL. Then, tpOrderType or slOrderType must be `Market`</li><li>`Partial`: partial position tp/sl (as there is no size option, so it will create tp/sl orders with the qty you actually fill). Limit TP/SL order are supported. Note: When create limit tp/sl, tpslMode is **required** and it must be `Partial`</li></ul>Valid for `linear`|
|> tpLimitPrice |false |string |The limit order price when take profit price is triggered <ul><li>`linear`: only works when tpslMode=Partial and tpOrderType=Limit</li><li>Spot(UTA): it is required when the order has `takeProfit` and `tpOrderType=Limit`</li></ul>|
|> slLimitPrice |false |string |The limit order price when stop loss price is triggered<ul><li>`linear`: only works when tpslMode=Partial and slOrderType=Limit</li><li>Spot(UTA): it is required when the order has `stopLoss` and `slOrderType=Limit`</li></ul>|
|> tpOrderType |false |string |The order type when take profit is triggered <ul><li>`linear`: `Market`(default), `Limit`. For tpslMode=Full, it only supports tpOrderType=Market</li><li>Spot(UTA): <br/>`Market`: when you set "takeProfit", <br/>`Limit`: when you set "takeProfit" and "tpLimitPrice" </li></ul>|
|> slOrderType |false |string |The order type when stop loss is triggered <ul><li>`linear`: `Market`(default), `Limit`. For tpslMode=Full, it only supports slOrderType=Market</li><li>Spot(UTA): <br/>`Market`: when you set "stopLoss", <br/>`Limit`: when you set "stopLoss" and "slLimitPrice" </li></ul>|
