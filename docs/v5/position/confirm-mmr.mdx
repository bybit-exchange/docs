It is only applicable when the user is marked as only reducing positions (please see the isReduceOnly field in
the [Get Position Info](/v5/position#) interface). After the user actively adjusts the risk level, this interface
is called to try to calculate the adjusted risk level, and if it passes (retCode=0), the system will remove the position reduceOnly mark.
You are recommended to call [Get Position Info](/v5/position#) to check `isReduceOnly` field.
> **Unified account covers: USDT perpetual / USDC contract / Inverse contract**<br/>
> **Classic account covers: USDT perpetual / Inverse contract**

### HTTP Request
POST `/v5/position/confirm-pending-mmr`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |Product type <ul><li>Unified account: `linear`, `inverse`</li><li>Classic account: `linear`, `inverse`</li></ul>|
|symbol |**true** |string |Symbol name|

### Response Parameters
None
