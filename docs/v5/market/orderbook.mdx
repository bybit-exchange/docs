Query for orderbook depth data.
> **Covers: Spot / USDT perpetual / USDC contract / Inverse contract / Option**

* Contract: 500-level of orderbook data
* Spot: 50-level of orderbook data
* Option: 25-level of orderbook data

:::info
The response is in the snapshot format.
:::

### HTTP Request
GET `/v5/market/orderbook`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |Product type. `spot`, `linear`, `inverse`, `option` |
|symbol |**true** |string |Symbol name|
|limit |false |integer |Limit size for each bid and ask<ul><li>`spot`: [`1`, `200`]. Default: `1`.</li><li>`linear`&`inverse`: [`1`, `500`]. Default: `25`.</li><li>`option`: [`1`, `25`]. Default: `1`.</li></ul>|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|s |string |Symbol name |
|b |array |Bid, buyer. Sort by price desc |
|> b[0] |string |Bid price |
|> b[1] |string |Bid size |
|a |array |Ask, seller. Order by price asc |
|> a[0] |string |Ask price |
|> a[1] |string |Ask size |
|ts |integer |The timestamp (ms) that the system generates the data |
|u |integer |Update ID, is always in sequence<ul><li>For contract, it is corresponding to `u` in the wss 500-level orderbook</li><li>For spot, it is corresponding to `u` in the wss 200-level orderbook</li></ul> |
|seq |integer |Cross sequence <li>You can use this field to compare different levels orderbook data, and for the smaller seq, then it means the data is generated earlier. </li><li>Option does not have this field currently</li> |

<Tabs groupId="programming-languages">
<TabItem value="http" label="HTTP">


</TabItem>
<TabItem value="python" label="Python">

```python
from pybit.unified_trading import HTTP
session = HTTP(testnet=True)
print(session.get_orderbook(
    category="linear",
    symbol="BTCUSDT",
))
```

</TabItem>
<TabItem value="java" label="Java">

```java
import com.bybit.api.client.domain.CategoryType;
import com.bybit.api.client.domain.market.*;
import com.bybit.api.client.domain.market.request.MarketDataRequest;
import com.bybit.api.client.service.BybitApiClientFactory;
var client = BybitApiClientFactory.newInstance().newAsyncMarketDataRestClient();
var orderbookRequest = MarketDataRequest.builder().category(CategoryType.SPOT).symbol("BTCUSDT").build();
client.getMarketOrderBook(orderbookRequest,System.out::println);
```

</TabItem>
