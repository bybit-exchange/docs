Obtain transaction information. **Up to 50 requests per second ** 
:::info
<li>Field query priority: rfqId > rfqLinkId  quoteId > quoteLinkId</li>
:::


### HTTP Request
GET `/v5/rfq/trade-list`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|rfqId |**false** |string|Inquiry ID|
|rfqLinkId |**false**  |string |Custom ID for inquiry form; specify rfqLinkId to only check the last 3 months|
|quoteId |**false** |string|Quotation ID|
|quoteLinkId |**false**  |string |Quotation custom ID; specifying quoteLinkId can only check the last 3 months|
|traderType |**false**  |string |Trader type, `quote` , `request` , default `quote`<li>`Request` : Inquirer, inquire about the successful transaction of their own inquiry </li><li>`Quote` : Quoters, check their own quotes for successful transactions</li>|
|status |string |Status : `Filled` `Failed` |
|limit |**false** |integer|Return the number of items, maximum 100 items, default 50 items|
|cursor |**false** |string|For page turning mark, please use the returned cursor, use the returned original data when signing, and carry out URLEncode when sending request|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|result |Object ||
|> cursor |string |page turning mark|
|> list |Array |the array of rfq|
|>> rfqId |string |Inquiry ID|
|>> rfqLinkId |string |Custom ID for inquiry form, sensitive information for customers, will not be disclosed, return "to the quoting party. |
|>> quoteId |string |Return the completed inquiry form and the executed quotation id.|
|>> quoteLinkId |string |Quotation custom ID, sensitive information for customers, will not be disclosed, return "to the inquiring party. |
|>> quoteSide |string |Return of completed inquiry, execution of quotation direction, `Buy` or `Sell`|
|>> strategyType |string |Inquiry label|
|>>status |string |Status : `Filled` `Failed`|
|>> rfqDeskCode |string |The unique identification code of the inquiry party, which is not visible when anonymous is set to `true` during inquiry |
|>> quoteDeskCode |string |The unique identification code of the quotation party, which is not visible when anonymous is set to `true` during quotation |
|>> createdAt |string |Time(ms) when the trade is created in epoch, such as 1650380963 |
|>> updatedAt |string |Time(ms) when the trade is updated in epoch, such as 1650380964 |
|>> legs |Array of objects |Combination transaction |
|>>> category |string |category. Valid values include: `linear`, `option` and `spot` |
|>>> orderId |string |bybit order id |
|>>> symbol |string |The unique instrument ID|
|>>> side |string |Direction, valid values are `Buy` and `Sell`|
|>>> price |string |Execution price|
|>>> qty |string |Number of executions|
|>>> markPrice |string |The futures markPrice at the time of transaction, the spot is indexPrice, and the option is the markPrice of the underlying Price.|
