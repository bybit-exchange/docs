RFQ Config. **Up to 50 requests** per second.
:::info
Query the information of the quoting party that can participate in the transaction and your own deskCode and other configuration information
:::

### HTTP Request
GET `/v5/rfq/config`

### Request Parameters
None

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|result |array |Order ID |
|list |Object ||
|> deskCode |string |Own deskCode, unique identification code |
|> maxLegs |integer |Maximum number of legs |
|> maxLP |integer |The maximum number of LPs selected in the inquiry form |
|> maxActiveRfq |integer |The maximum number of unfinished inquiry orders allowed by a user |
|> rfqExpireTime |integer |Inquiry expiration time, minutes |
|> minLimitQtySpotOrder |integer |Spot minimum order quantity multiplier |
|>minLimitQtyContractOrder |integer |Contract minimum order quantity multiplier |
|> minLimitQtyOptionOrder |integer |Option minimum order multiplier |
|> strategyTypes |array |Product strategy |
|>> strategyName |string |Policy name |
|> strategyTypes |array |Information on the bidder who can participate in the transaction |
|>> traderName |string |Name of the bidder |
|>> deskCode |string |The unique identification code of the quotation party is publicly visible; the relevant interfaces of quotation and inquiry use this code to represent the quotation party |
|>> type |string |Quoter type. `LP` is an automated market maker connected via API, null means a normal quote party |
