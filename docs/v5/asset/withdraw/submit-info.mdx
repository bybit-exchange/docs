Submit the originator's compliance information when a deposit triggers a Travel Rule review. After submission, the vendor completes the review and returns the resulting status.

:::info
Call this endpoint when a deposit record hits the Travel Rule compliance policy and the current `travel_rule_status` is `1` (pending counterparty info submission).
:::

### HTTP Request
<APIEndpoint method="POST" url="/v5/asset/travel-rule/deposit/submit" />

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|depositId |**true** |integer |Deposit record ID obtained from the deposit query API|
|subAccountId |false |integer |Broker scenario: target sub-account UID. Pass `0` or omit to use the current account |
|questionnaire |**true** |string |Travel Rule questionnaire info as a JSON string. Max 16384 bytes. Structure varies by compliance zone. See [Questionnaire](questionnaire) |

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|travelRuleStatus |integer|Travel Rule review status <ul><li>`0`: Approved — review passed, proceed with subsequent flow</li><li>`1`: CollectInfo — counterparty info required, re-submit questionnaire</li><li>`2`: Pending — under review, poll the deposit query endpoint</li><li>`3`: Rejected — rejected or failed (including cancelled)</li></ul> |
