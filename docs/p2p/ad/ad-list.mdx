### HTTP Request
POST `/v5/p2p/item/personal/list`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|itemId|false|string|ad's id|
|status|false|string|`1`: Sold Out; `2`: Available|
|side|false|string|ad's for buying token or for selling token. `0`: buy; `1`: sell|
|tokenId|false|string|Token id. e.g. USDT,ETH,BTC|
|page|false|string|Page number. Default Value is 1|
|size|false|string|Page size. Default Value is 10|
|currencyId|false|string|Currency id e.g HKD,USD,EUR|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|count|integer|Total page count|
|hiddenflag|boolean|Are any ads hidden? `true`: There are ads that are hidden, `false`: No ads are hidden|
|items|array&lt;object&gt;|ads Information. see Items Fields|
|> id|string|ads id|
|> accountId|string|accId|
|> userId|string|user_id|
|> nickName|string|Nick name|
|> tokenId|string|token_id|
|> currencyId|string|currency_id|
|> side|integer|ad for buying token or for selling token, 0 - buy, 1 - sell|
|> priceType|integer|Ad Pricing Exchange Rate Model, 0 - Fixed rate, 1 - Floating rate|
|> price|string|ad price|
|> premium|string|Floating ratio with current exchange rate|
|> lastQuantity|string|Remaining tradable quantity of ads|
|> quantity|string|Number of ads|
|> frozenQuantity|string|Number of ads locked for ongoing transactions|
|> executedQuantity|string|Number of ads sold|
|> minAmount|string|Min transaction amount|
|> maxAmount|string|Max transaction amount|
|> remark|string|Transaction Description|
|> status|integer|Ads Status: 10 - Online; 20 - Offline; 30 - Completed|
|> createDate|string|creation time(timestamp)|
|> payments|array&lt;string&gt;|Payment method type id|
|> tradingPreferenceSet|object|Trading Preferences|
|>> hasUnPostAd|string|Whether the counterparty must not have posted any advertisements: 0:unneed,1:need|
|>> isKyc|string|Is it necessary for the counterparty to complete identity authentication?: 0:unneed,1:need|
|>> isEmail|string|Is it necessary for the counterparty to bind an email address?:0:unneed,1:need|
|>> isMobile|string|Is it necessary for the counterparty to bind an telphone?:0:unneed,1:need|
|>> hasRegisterTime|string|Is the registration time required to be no less than {} days?0:unneed,1:need|
|>> registerTimeThreshold|string|Registration time threshold(Unit: Day)|
|>> orderFinishNumberDay30|string|Limit on number of completed orders in the last 30 days|
|>> completeRateDay30|string|Completion rate limit in the last 30 days|
|>> nationalLimit|string|Restricted KYC Countries;IOS 3 CODE|
|>> hasOrderFinishNumberDay30|string|Is it necessary to have no less than {} orders in the last 30 days? 0:unneed,1:need|
|>> hasCompleteRateDay30|string|Is it necessary to have a completion rate of no less than {} in the last 30 days?:0:unneed,1:need|
|>> hasNationalLimit|string|Are country restrictions required;0:unneed,1:need|
|> updateDate|string|Update time (timestamp)|
|> feeRate|string|Fee Rate|
|> paymentPeriod|integer|payment duration|
|> itemType|string|ORIGIN - original P2P advertisement, generally refers to non-bulk advertisement, BULK - bulk advertisement|
|> paymentTerms|array&lt;object&gt;|Payment Methods|
|>> id|string|Payment method id|
|>> paymentType|string|Payment method type|
