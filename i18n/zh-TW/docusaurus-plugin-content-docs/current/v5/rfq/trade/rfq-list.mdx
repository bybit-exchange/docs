Obtain historical inquiry information. **Up to 50 requests per second ** 
:::info
<li>Obtain the information of the inquiry form sent or received by the user, query from the database, there is a delay</li><li> Pass both rfqId and rfqLinkId, rfqId shall prevail</li><li>Sort in reverse order according to the creation time of rfq and return it.</li>
:::


### HTTP Request
GET `/v5/rfq/rfq-list`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|rfqId |**false** |string|Inquiry ID|
|rfqLinkId |**false** |string|Custom ID for inquiry form; specify rfqLinkId to only check the last 3 months, traderType is quote, this field is invalid|
|traderType |**false**  |string |Trader type, `quoter` , `request` , default `request`<li>`Request` : Inquirer, to inquire about the inquiry they have issued</li><li>`Quoter` : Quote the quotation received by oneself</li>|
|status |string |Status of the inquiry form:`Active` `Canceled` `Filled` `Expired` `Failed` |
|limit |**false** |integer|Return the number of items, maximum 100 items, default 50 items|
|cursor |**false** |string|For page turning mark, please use the returned cursor, use the returned original data when signing, and carry out URLEncode when sending request|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|result |array |Object|
|> cursor |string |page turning mark|
|> list |Array |the array of rfq|
|>> rfqId |string |Inquiry ID|
|>> rfqLinkId |string |Custom ID for inquiry form, sensitive information for customers, will not be disclosed, return "to the quoting party. |
|>> counterparties |Array of srings |List of bidders |
|>> expiresAt |string |The expiration time of the inquiry form, in milliseconds format of Unix timestamp |
|>> strategyType |string |Inquiry label |
|>> status |string |Status of the inquiry form:`Active` `PendingFill` `Canceled` `Filled` `Expired` `Failed` |
|>> deskCode |string |The unique identification code of the inquiry party, which is not visible when anonymous is set to `true` during inquiry |
|>> createdAt |string |Time(ms) when the trade is created in epoch, such as 1650380963 |
|>> updatedAt |string |Time(ms) when the trade is updated in epoch, such as 1650380964 |
|>> legs |Array of objects |Combination transaction |
|>>> category |string |category. Valid values include: "linear", "option" and "spot" |
|>>> symbol |string |The unique instrument ID|
|>>> side |string |Inquiry direction: Valid values are `Buy` and `Sell` . |
|>>> qty |string |Order quantity of the instrument. |
