**基本概念**

1. **報價請求（RFQs）** - 報價請求是由詢價方發送給報價方的文件，包含一個或多個產品及其數量，詢價方希望進行交易。<br/>  
2. **報價單** - 報價單是報價方針對詢價單所提供的報價。<br/>  
3. **交易** - 當詢價方接受並執行報價方的報價時，交易即完成。<br/>  

**基本工作流程**

1. 詢價方創建報價請求（RFQ），並選擇希望接收RFQ的報價方。<br/>  
2. 不同的報價方針對此詢價發送報價。<br/>  
3. 詢價方選擇執行最佳報價以生成交易。Blocktrade接收交易並進行結算。<br/>  
4. 詢價方和報價方收到交易執行確認。<br/>  
5. 交易詳情會發布在公開市場數據頻道（不包含交易方信息）。<br/>  

**詢價方視角（OpenAPI）**  
1. 詢價方使用 [/v5/rfq/create-rfq](../rfq/trade/create-rfq#) 創建詢價。詢價方可以通過 [/v5/market/instruments-info](../market/instrument#) 查詢可詢價的產品信息，並通過 [/v5/rfq/config query](../rfq/trade/rfq-config#) 選擇報價方信息。<br/>  
2. 詢價方可以隨時通過 [/v5/rfq/cancel-rfq](../rfq/trade/cancel-rfq#) 取消有效的詢價。<br/>  
3. 報價方（若為詢價方選定的報價方之一）將在 [rfq.open.rfqs](../rfq/websocket/private/inquiry#) 推送頻道接收到詢價信息，並可進行相應報價。<br/>  
4. 詢價方在 [rfq.open.quotes](../rfq/websocket/private/quote#) 推送頻道接收到報價信息後，可以選擇最佳報價並通過 [/v5/rfq/execute-quote](../rfq/trade/execute-quote#) 執行報價。<br/>  
5. 詢價方將在 [rfq.open.trades](../rfq/websocket/private/transaction#) 和 [rfq.open.rfqs](../rfq/websocket/private/inquiry#) 推送頻道收到交易成功執行的確認。<br/>  
6. 詢價方也會在 [rfq.open.public.trades](../rfq/websocket/public/public-transaction#) 推送頻道收到此及其他大宗交易的確認。<br/>  

**報價方視角（OpenAPI）**  
1. 當新的報價請求發出且報價方為選定的報價方之一時，報價方將在 [rfq.open.rfqs](../rfq/websocket/private/inquiry#) 推送頻道收到此請求信息。<br/>  
2. 報價方創建報價並通過 [/v5/rfq/create-quote](../rfq/trade/create-quote#) 發送。<br/>  
3. 報價方可以隨時通過 [/v5/rfq/cancel-quote](../rfq/trade/cancel-quote#) 取消有效的報價。<br/>  
4. 詢價方選擇執行最佳報價。<br/>  
5. 報價方通過 [rfq.open.quotes](../rfq/websocket/private/quote#) 推送頻道接收其報價的狀態更新。<br/>  
6. 報價方將在 [rfq.open.trades](../rfq/websocket/private/transaction#) 和 [rfq.open.quotes](../rfq/websocket/private/quote#) 推送頻道收到其報價成功執行的確認。<br/>  
7. 報價方也會在 [rfq.open.public.trades](../rfq/websocket/public/public-transaction#) 推送頻道收到此交易及其他大宗交易的確認。<br/>
