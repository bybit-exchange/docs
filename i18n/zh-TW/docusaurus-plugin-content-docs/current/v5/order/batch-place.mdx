> **覆蓋範圍: 現貨 (統一帳戶) / 期權 (統一帳戶) / USDT永續、USDC永續、USDC交割 (UTA Pro)**

:::tip
該接口支持批量創建委託單
- 下單時需確保帳戶內有足夠的資金。一旦下單，根據訂單所需資金，您的帳戶資金將在訂單生命週期內凍結相應額度，被凍結的資金額度取決於訂單屬性。
- 每個請求包含的訂單數最大是: 20筆(期权), 10筆(期貨), 10筆(現貨)，返回的數據列表中分成兩個list，訂單創建的列表和創建結果的信息返回，兩個list的訂單的序列是完全保持一致的。
:::

:::info
- 當category=`linear` 或者 `spot`時, 請從[這裡](../rate-limit#不同vippro等級的接口限頻規則)查閱其API限頻說明

- **風控限制提示:**<br/>
Bybit 將針對您的 API 請求進行統計監控，當單日 (UTC 0点 - UTC 24点) 單帳號（母帳號和子帳號整體運算）訂單總数超過一定上限，平台將保留提醒、警告，以及進行必要性限制的權利。
使用API的客戶預設接受本條款並負有配合調整的義務。
:::

### HTTP請求
POST `/v5/order/create-batch`

### 請求參數
| 參數 | 是否必需 | 類型 | 說明|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |產品類型. `linear`, `option`, `spot` |
|request |**true** |array |Object |
|> symbol |**true** |string |合約名稱|
|> isLeverage |false |integer |是否借貸. 僅**統一帳戶的現貨交易**有效. `0`(default): 否，則是幣幣訂單, `1`: 是，則是槓桿訂單|
|> side |**true** |string |方向. `Buy`, `Sell`|
|> [orderType](../enum#ordertype) |**true** |string |訂單類型. `Market`, `Limit`|
|> qty |**true** |string |訂單數量 訂單數量 <ul><li>統一帳戶<ul><li>現貨: 可以通過設置`marketUnit`來表示市價單qty的單位, 市價買單默認是`quoteCoin`, 市價賣單默認是`baseCoin`</li><li>期貨和期權: 總是以base coin作為qty的單位</li></ul></li><li>期貨: 如果傳入`qty`="0"以及`reduceOnly`="true", 則可以全部平掉對應合約的倉位</li></ul>|
|> price |false |string |訂單價格 <ul><li>市價單將會忽視該字段</li><li>請通過該[接口](../market/instrument#response-parameters)確認最低價格和精度要求</li><li>如果有持倉, 確保價格優於強平價格</li></ul>|
|> marketUnit |false |string |**統一帳戶現貨交易**創建市價單時給入參`qty`指定的單位<li>`baseCoin`: 比如, 買BTCUSDT, 則"qty"的單位是BTC</li><li>`quoteCoin`: 比如, 賣BTCUSDT, 則"qty"的單位是USDT</li>|
|> triggerDirection |false |integer |條件單參數. 用於辨別期望的方向. <ul><li>`1`: 當市場價上漲到了`triggerPrice`時觸發條件單</li><li>`2`: 當市場價下跌到了`triggerPrice`時觸發條件單</li></ul>僅`linear`有效|
|> orderFilter |false |string |指定訂單品種. `Order`: 普通單,`tpslOrder`: 止盈止損單,`StopOrder`: 條件單. 若不傳, 默認`Order`<br/>僅對現貨有效|
|> triggerPrice |false |string |<ul><li>對於期貨, 是條件單觸發價格參數. 若您希望市場價是要上升後觸發, 確保:<br/> _triggerPrice > 市場價格_<br/>否則, _triggerPrice < 市場價格_</li><li>對於現貨, 這是下止盈止損單(orderFilter=tpslOrder)或者條件單(orderFilter=stopOrder)的觸發價格參數</li></ul>|
|> [triggerBy](../enum#triggerby) |false |string |條件單參數. 觸發價格類型. `LastPrice`, `IndexPrice`, `MarkPrice`<br/>僅`linear`有效|
|> orderIv |false |string |隱含波動率. 僅`option`有效. 按照實際值傳入, e.g., 對於10%, 則傳入0.1. `orderIv`比`price`有更高的優先級 |
|> [timeInForce](../enum#timeinforce) |false |string |<a href="https://www.bybit.com/zh-TW/help-center/bybitHC_Article?language=zh_TW&id=000001044">訂單執行策略</a><ul><li>市價單，系統直接使用`IOC`</li><li>若不傳，默認使用`GTC`</li> </ul>|
|> [positionIdx](../enum#positionidx) |false |integer |倉位標識, 用戶不同倉位模式. 該字段對於雙向持倉模式(USDT永續支持雙向模式)是**必傳**:<ul><li>`0`: 單向持倉</li><li>`1`: 買側雙向持倉</li><li>`2`: 賣側雙向持倉</li></ul>僅`linear`有效|
|> orderLinkId |false |string |用戶自定義訂單Id. category=`option`時，該參數必傳|
|> takeProfit |false |string |止盈價格, 对`linear`和`spot`有效 |
|> stopLoss |false |string |止損價格, 对`linear`和`spot`有效 |
|> [tpTriggerBy](../enum#triggerby) |false |string |觸發止盈的價格類型. `MarkPrice`, `IndexPrice`, 默認:`LastPrice`<br/>僅`linear`有效|
|> [slTriggerBy](../enum#triggerby) |false |string |觸發止損的價格類型. `MarkPrice`, `IndexPrice`, 默認:`LastPrice`<br/>僅`linear`有效|
|> reduceOnly |false |boolean |<a href="https://www.bybit.com/zh-TW/help-center/bybitHC_Article?language=zh_TW&id=000001047">什麼是只減倉?</a> `true` 將這筆訂單設為只減倉 <ul><li>當減倉時, reduceOnly=true**必傳**</li><li>只減倉單的止盈止損不生效</li></ul> |
|> closeOnTrigger |false |boolean |<a href="https://www.bybit.com/zh-TW/help-center/bybitHC_Article?language=zh_TW&id=000001050">什麼是觸發後平倉委託?</a>此選項可以確保您的止損單被用於減倉（平倉）而非加倉，並且在可用保證金不足的情況下，取消其他委託，騰出保證金以確保平倉委託的執行.<br/>僅`linear`有效 |
|> [smpType](../enum#smptype) |false |string |Smp執行類型. [什麼是SMP?](../smp)|
|> mmp |false |boolean |做市商保護. 僅`option`有效. `true` 表示該訂單是做市商保護訂單. [什麼是做市商保護?](../account/set-mmp)|
|> tpslMode |false |string |止盈止損模式 <ul><li>`Full`: 全部倉位止盈止損.  此時, tpOrderType或者slOrderType必須傳`Market`</li><li>`Partial`: 部分倉位止盈止損. 支持創建限價止盈止損. 注意: 創建限價止盈止損時, tpslMode**必傳**且為Partial</li></ul>僅`linear`有效|
|> tpLimitPrice |false |string |觸發止盈後轉換為限價單的價格 <ul><li>`linear`: 僅作用於當tpslMode=Partial以及tpOrderType=Limit時</li><li>`spot`(UTA): 參數必傳當創建訂單時帶了`takeProfit` 和 `tpOrderType=Limit`</li></ul>|
|> slLimitPrice |false |string |觸發止損後轉換為限價單的價格 <ul><li>`linear`: 僅作用於當tpslMode=Partial以及slOrderType=Limit時</li><li>`spot`(UTA): 參數必傳當創建訂單時帶了`stopLoss` 和 `slOrderType=Limit`</li></ul>|
|> tpOrderType |false |string |止盈觸發後的訂單類型 <ul><li>`linear`: `Market`(默認), `Limit`. 對於tpslMode=Full, 僅支持tpOrderType=Market</li><li>`spot`(UTA): <br/>`Market`: 當帶了參數"takeProfit", `Limit`: 當帶了參數"takeProfit" 和 "tpLimitPrice" </li></ul>|
|> slOrderType |false |string |止損觸發後的訂單類型 <ul><li>`linear`: `Market`(默認), `Limit`. 對於tpslMode=Full, 僅支持slOrderType=Market</li><li>`spot`(UTA): <br/>`Market`: 當帶了參數"stopLoss", `Limit`: 當帶了參數"stopLoss" 和 "slLimitPrice" </li></ul>|
