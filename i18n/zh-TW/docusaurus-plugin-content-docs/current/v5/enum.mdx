### orderStatus
_活動態_
* `New`&emsp;訂單成功下達
* `PartiallyFilled`&emsp;部分成交
* `Untriggered`&emsp;條件單未觸發

_終態_
* `Rejected`&emsp;訂單被拒絕
* `PartiallyFilledCanceled`&emsp;僅現貨存在該枚舉值, 訂單部分成交且已取消
* `Filled`&emsp;完全成交
* `Cancelled`&emsp;期貨交易，當訂單是該狀態時，是可能存在部分成交的; 經典帳戶的現貨盈止損單、條件單、OCO訂單觸發前取消
* `Triggered`&emsp;已觸發, 條件單從未觸發到變成New的一個中間態
* `Deactivated`&emsp;統一帳戶下期貨、現貨的盈止損單、條件單、OCO訂單觸發前取消

### timeInForce
* `GTC`&emsp;一直有效至取消
* `IOC`&emsp;立即成交或取消
* `FOK`&emsp;完全成交或取消
* <a href="https://www.bybit.com/zh-TW/help-center/bybitHC_Article?language=zh_TW&id=000001051">PostOnly</a>&emsp;被動委託
* <a href="https://www.bybit.com/en/help-center/article/Retail-Price-Improvement-RPI-Order">RPI(Retail Price Improvement)</a>&emsp;特性:<ul><li><b>獨家匹配</b>: 僅匹配非演算法用戶；不執行來自 Open API 的訂單.</li>
<li><b>Post-Only 機制</b>: 充當掛單，增加流動性</li>
<li><b>較低優先順序</b>: 在相同價格水準的非 RPI 訂單之後執行</li>
<li><b>有限訪問</b>: 最初僅針對部分幣對的選定做市商.</li>
<li><b>訂單簿更新</b>: 從 API 中排除，但顯示在 GUI 上.</li></ul>

### curAuctionPhase
* `NotStarted`&emsp;盤前交易未開始
* `Finished`&emsp;盤前交易已結束
    * 當盤前合約最終無法轉為正式合約, 它將會被下架且phase="Finished"
* `CallAuction`&emsp;集合競價第一階段
    * 僅支持timeInForce=GTC, orderType=Limit的訂單
    * 不支持設置止盈止損; 不支持掛條件單
    * 該階段不支持改單
    * 訂單價格區間: [[preOpenPrice](../v5/market/tickers) x 0.5, [maxPrice](../v5/market/instrument)]
* `CallAuctionNoCancel`&emsp;集合競價第二階段
    * 僅支持timeInForce=GTC, orderType=Limit的訂單
    * 不支持設置止盈止損; 不支持掛條件單
    * 該階段不支持改單和撤單
    * 訂單價格區間: 買單 [[lastPrice](../v5/market/tickers) x 0.5, [markPrice](../v5/market/tickers) x 1.1], 賣單 [[markPrice](../v5/market/tickers) x 0.9, [maxPrice](../v5/market/instrument)]
* `CrossMatching`&emsp;集合競價第三階段 - 撮合盤前交易
    * 該階段不支持掛單, 改單和撤單
    * 該階段會產生k線數據
* `ContinuousTrading`&emsp;連續競價階段
    * 該階段對訂單行為沒有限制
    * 該階段會產生訂單簿、最近成交數據

### 現貨交易手續費幣種說明
以BTCUSDT為例:

- maker費率是否為正?
  - TRUE
      - 方向 = Buy -> 交易幣種 (BTC)
      - 方向 = Sell -> 報價幣種 (USDT)
  - FALSE
    - 是否是maker單 = TRUE
        - 方向 = Buy -> 報價幣種 (USDT)
        - 方向 = Sell -> 交易幣種 (BTC)
    - 是否是maker單 = FALSE
        - 方向 = Buy -> 交易幣種 (BTC)
        - 方向 = Sell -> 報價幣種 (USDT)
