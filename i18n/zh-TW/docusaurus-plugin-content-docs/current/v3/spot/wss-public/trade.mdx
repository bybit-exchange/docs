<p>逐筆交易推送每一筆成交的信息。成交，或者說交易的定義是僅有一個吃單者與一個掛單者相互交易。</p>
<p>變量「v」可以理解成一個交易ID。這個變量是全局遞增的並且獨特的。例如：假設過去5秒有3筆交易發生，分別是ETHUSDT、BTCUSDT、BHTBTC。它們的「v」會為連續的值（112，113，114）。</p>
<p><b>推送頻率</b>: 實時</p>

**Topic:** `trade.{symbol}`

| 參數 | 類型 | 說明|
|:----- |:-----|----- |
|topic |string |Topic名 |
|ts |number |消息被推送出來的時間戳 (毫秒) |
|type |string |數據類型. `snapshot`|
|data |array |Object |
|> v | string | 成交ID |
|> t | number | 撮合引擎撮合时间戳 (毫秒) |
|> p | string | 成交價格 |
|> q | string | 成交數量 |
|> m | boolean | 交易類型標識. 0：幣幣成交. 1：Paradigm 大宗交易 |

### 訂閱示例

```javascript
{
    "req_id": "trade0001", //optional
    "op": "subscribe",
    "args": [
        "trade.BTCUSDT"
    ]
}
```

### 推送示例

```json
{
    "topic": "trade.BTCUSDT",
    "ts": 1673437259397,
    "type": "snapshot",
    "data": {
        "v": "2290000000036455162",
        "t": 1673437259395,
        "p": "17440",
        "q": "0.0002",
        "m": false,
        "type": "0"
    }
}
```
