風險限額會限制你在不同的保證金要求下的可持倉的最大倉位數量，如果要持有更多的倉位，需要更多的保證金要求。本接口可以設置單個倉位的風險限額，
下單時如果超過當前的風險限額，則會被拒絕。點擊<a href="https://www.bybit.com/zh-TW/help-center/bybitHC_Article?language=zh_TW&id=000001164">這裡</a>了解更多關於風險限額.

:::tip
您可以獲取通過該[接口](../public/risk-limit#)查詢到風險限額信息.
:::

### HTTP 請求
POST `/unified/v3/private/position/set-risk-limit`

### 請求參數
| 參數 | 是否必需 | 類型 | 說明|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |產品類型. `linear`|
|symbol |**true** |string |合約名稱|
|riskId |**true** |integer |風險限額Id|
|positionIdx |false |integer |倉位標識，用於標識不同倉位. `0`: 單倉模式, 統一保證金僅有單倉模式|

### 響應參數
無

### 請求示例

```http
POST /unified/v3/private/position/set-risk-limit HTTP/1.1
Host: api-testnet.bybit.com
X-BAPI-SIGN: XXXXX
X-BAPI-API-KEY: XXXXX
X-BAPI-TIMESTAMP: 1672016123862
X-BAPI-RECV-WINDOW: 5000
Content-Type: application/json

{
    "category": "linear",
    "symbol": "BTCUSDT",
    "riskId": 2,
    "positionIdx": 0
}
```
